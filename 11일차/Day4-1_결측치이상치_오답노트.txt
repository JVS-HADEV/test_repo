Q1. 컬럼마다 IQR 이상치 제거를 반복하면 데이터가 3개만 남는 이유는?

내 답변:
    컬럼별로 이상치가 있는 행이 오버랩이 안 돼어서 행제거가 많이 됨

채점: 부분 정답

해설:
    - 방향은 맞음. 핵심은 "컬럼마다 이상치인 행이 다르다"는 점.
    - 순차적으로 행을 지우면 "모든 컬럼의 IQR 범위를 통과한 행(교집합)"만 남음.
    - 센서 컬럼 563개 × 컬럼당 약 5% 이상치 → 행마다 평균 17개 컬럼에서 이상치.
    - 한 컬럼이라도 범위 밖이면 제거 → 1567행 → 3행.
    - "오버랩이 안 된다"보다 "교집합이 너무 작아진다"가 더 정확한 표현.
    - 추가: 순차 제거 시 매번 quantile을 다시 계산하면 경계도 바뀌어 더 줄어들 수 있음.


Q2. IQR 이상치 판별에서 하한·상한 공식은?

내 답변:
    하한 = Q1 - 1.5*IQR, 상한 = Q3 + 1.5*IQR

채점: 정답

해설:
    - IQR = Q3 - Q1 (사분위 범위)
    - 하한(lower) = Q1 - 1.5 × IQR
    - 상한(upper) = Q3 + 1.5 × IQR
    - 이 범위 밖의 값을 이상치(outlier)로 봄.
    - boxplot의 수염(whisker) 끝과 같은 기준.


Q3. 3행만 남는 문제를 해결하는 더 나은 이상치 처리 방법은?

내 답변:
    결측치가 너무 많은 열 삭제

채점: 오답 (맥락 불일치)

해설:
    - "결측치가 많은 열 삭제"는 Section 1 결측치 처리 방법.
    - 이상치 문제(행이 과도하게 줄어듦)와는 별개.
    - 이상치 대안 예시 (자세한 설명은 [3], [4] 참고):
        (1) 윈저화(Winsorization): 행 유지, 경계값으로 클리핑 → [2]
        (2) 투표 방식: N개 이상 컬럼에서 이상치일 때만 행 제거 → [3]
        (3) 핵심 변수만 IQR 적용 (500+ 컬럼 전부 X) → [4]
    - 과제1에서는 (1) 윈저화를 적용.


실제 실행 결과(과제1):
    방법                  | 행 수  | 열 수

    원본 (data_imputed)   | 1,567  | 592
    순차 IQR 행제거       | 3      | 592

####################################################################
[2] 개념 보충 — 윈저화(Winsorization)
####################################################################

Q. 윈저화란 무엇인가?

해설:
    - 이상치를 "삭제"하지 않고, 경계값(lo/hi)으로 "잘라 붙이는" 방법.
    - pandas: Series.clip(lo, hi)
    - 행 수는 유지, 극단값만 완화.

    비유:
        행 제거  = 그 학생을 명단에서 제외
        윈저화   = 0점→10점, 100점→90점으로 기록만 수정 (학생은 유지)

    IQR 행 제거 vs 윈저화:
        |              | IQR 행 제거      | 윈저화           |
        |--------------|------------------|------------------|
        | 이상치 처리  | 해당 행 삭제     | 값을 경계로 조정 |
        | 행 수        | 줄어듦 (1567→3)  | 유지 (1567)      |
        | 정보 손실    | 행 전체 손실     | 극단값만 완화    |

    한계:
        - 이상치가 너무 많으면 경계값에 값이 몰려 분포 왜곡 가능.
        - 왜 이상치인지 모를 때는 원인 파악 후 처리하는 것이 바람직.


####################################################################
[3] 개념 보충 — 대안 1: 투표 방식 (Threshold 기반 행 제거)
####################################################################

Q. 윈저화 말고, 행을 일부 제거하면서도 데이터를 크게 잃지 않으려면?

해설:
    - 아이디어: "한 컬럼만 벗어남 = 무조건 제거"가 아니라,
      "여러 컬럼에서 동시에 이상치일 때만" 행을 제거.

    처리 순서 (개념):
        1) 모든(또는 일부) 컬럼에 IQR 규칙 적용 → 이상치 여부 True/False 표
        2) 행마다 이상치인 컬럼 개수를 합산
        3) 개수가 임계값 K 이상인 행만 제거 (예: K=10)

    비유:
        학생 1과목 0점 → 바로 퇴학 X
        5과목 이상 0점 → 비정상적으로 의심 → 퇴학(행 제거)

    이 데이터에 왜 맞나:
        - 행당 평균 17개 컬럼에서 IQR 이상치 → 1개만 튀는 건 흔함.
        - 많은 센서가 동시에 튀면 "공정 이상" 가능성이 큼.
        - K를 적절히 두면 1567행 대부분 유지 가능.

    장점:
        - 행 수를 과도하게 줄이지 않음.
        - "진짜 이상" 행에 집중할 수 있음.

    단점:
        - K(몇 개 이상이면 제거?)를 정해야 함 → 실험·도메인 지식 필요.
        - K가 너무 작으면 다시 행이 급감, 너무 크면 이상치를 거의 안 제거.

Q. 563개 센서 전부에 IQR을 쓰지 않고 이상치를 다루려면?

해설:
    - 아이디어: 모든 변수가 동등하게 중요하지 않음.
      Pass/Fail 예측·공정 이해에 중요한 센서만 골라 IQR 적용.

    변수 선정 예시:
        - EDA: Pass/Fail과 상관 높은 센서
        - 모델: feature_importances_ 상위 변수
        - 도메인: 공정 담당자가 지정한 핵심 센서 5~20개

    처리 방식:
        - 선정된 10개 변수에만 IQR 행 제거(또는 윈저화) 적용
        - 나머지 550+ 변수는 그대로 두거나 별도 처리

    비유:
        건강검진에서 혈압·혈당만 기준 초과 시 재검사.
        몸속 500개 지표 전부 한 번에 탈락시키지 않음.

    장점:
        - 적용 변수 수가 줄어 "교집합 붕괴" 문제 완화.
        - 해석이 쉬움 ("이 센서들 기준으로 이상 공정 제거").

    단점:
        - 어떤 변수가 핵심인지 선정이 필요 (주관·분석 의존).
        - 빼둔 변수의 이상치는 그대로 남을 수 있음.

[5] 개념 보충 — 윈저화 후 quantile(분위수) 변동
####################################################################

Q. 윈저화를 한 이후에 quantile 값이 변동되지는 않아?

채점: 좋은 질문 (개념 이해)

해설:
    - 맞음. 윈저화 후에는 분위수가 "변할 수 있다".
      (분포 자체가 바뀌므로 Q1, Q3도 달라질 수 있음)

    - 다만 항상 크게 변하는 것은 아님:
        | 상황                              | 분위수 변화        |
        |-----------------------------------|--------------------|
        | 꼬리 극단값 몇 개만 클리핑        | 거의 안 변함 (흔함)|
        | 이상치가 많아 lo/hi에 값이 몰림   | 변할 수 있음       |
        | Q1·Q3 근처 값들이 잘림            | Q1, Q3 자체가 밀림 |

    - 과제1 코드의 처리 순서:
        1) 원본 data_imputed로 Q1, Q3, lo, hi 계산
        2) 그 경계로 clip (1회만)
        3) 윈저화된 데이터로 분위수를 다시 계산하지 않음

    - 정리:
        * 경계(lo, hi)는 "윈저화 전 원본" 기준
        * 윈저화 후 분포는 달라질 수 있음
        * 반복(윈저화 → 재계산 → 윈저화)하면 경계가 점점 좁아질 수 있음
          → 일반적으로 1회 윈저화로 충분

####################################################################
[7] 자주 헷갈리는 포인트
####################################################################

1. 결측치 처리 vs 이상치 처리
    - 결측치: 비어 있는 값 (NaN) → 삭제/대체/열 삭제
    - 이상치: 범위 밖의 값 → 삭제/클리핑/플래그

2. "컬럼별 이상치 개수 세기" vs "컬럼별 이상치 행 제거"
    - 개수만 세면: 데이터 행 수 유지 (EDA용)
    - 매 컬럼마다 행 제거: 교집합으로 행 수 급감 (위험)

3. 윈저화 경계는 언제 기준?
    - clip 직전, 원본 데이터의 quantile로 계산한 lo/hi
    - clip 이후 quantile을 다시 쓰지 않음 (1회 처리)

4. IQR 공식
    - IQR = Q3 - Q1 (Q3 + Q1 이 아님!)
    - 1.5배는 경험적 표준 (boxplot whisker와 동일)

5. 다변량(고차원) 데이터에서 이상치
    - 변수가 많을수록 "모든 변수에서 정상"인 행이 극소수
    - 행 삭제보다 값 조정(윈저화) 또는 변수 선택이 현실적
