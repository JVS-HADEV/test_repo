[1] [과제1] gamma 변화 Boundary 시각화 + nu vs gamma 역할
####################################################################

Q1. One-Class SVM은 어떤 데이터만으로 학습하고, boundary 밖은 무엇으로 판단하나?

내 답변:
    정상 데이터

채점: 정답 (보완 권장)

해설:
    - One-Class SVM(SVDD)은 정상(Normal, Y=1) 데이터만으로 학습.
    - 정상 데이터를 감싸는 최소 영역(경계)을 학습.
    - boundary 밖에 있는 점 → 이상치(Outlier)로 판단.
    - sklearn predict: 정상=1, 이상치=-1 (이 노트북의 Y 라벨과 동일 체계).


Q2. nu 파라미터를 크게 하면 boundary가 어떻게 변하나?

내 답변:
    작아짐

채점: 정답

해설:
    - nu ↑ → boundary 좁아짐(작아짐) → 더 많은 점을 이상치로 분류.
    - nu ↓ → boundary 넓어짐 → 더 많은 점을 정상으로 허용.
    - nu ≈ 학습 데이터 중 이상치로 허용할 비율의 상한.
    - SVDD의 C와 반대 방향으로 작동 (nu 클수록 더 엄격).

    실습 결과 (gamma=auto):
        nu 0.01~0.2 → F1 0.962
        nu 0.3      → F1 0.935
        nu 0.5      → F1 0.901
        → nu가 너무 크면 정상까지 이상치로 분류 → 성능 하락 가능.


Q3. RBF 커널에서 gamma를 크게 하면 boundary 모양이 어떻게 변하나?

내 답변:
    원

채점: 오답

해설:
    - gamma가 커진다고 "원"이 되는 것이 아님.
    - gamma ↑ → 각 데이터 점의 영향 범위가 좁아짐
      → boundary가 구불구불해지고, 점 주변으로 빽빽하게 감쌈 (과적합 경향).
    - gamma ↓ → 영향 범위가 넓어짐
      → boundary가 부드럽고 넓게 감쌈 (이때 원형에 가까운 매끄러운 형태 가능).

    | gamma | boundary 형태              | 경향        |
    |-------|----------------------------|-------------|
    | 작음  | 부드럽고 넓음              | underfit    |
    | 적당  | 데이터에 잘 맞는 균형      | 적정        |
    | 큼    | 복잡·구불구불, 점마다 움푹 | overfit     |

    과제1 실행 결과 예시 (nu=0.1 고정):
        gamma 0.01 → F1 0.778 (너무 넓어 이상치 놓침)
        gamma 0.10 → F1 0.951
        gamma 1.00 → F1 최고 근처 (적정)
        gamma 10   → F1 0.870 (Recall↑ but Precision↓, 과적합)


--------------------------------------------------------------------
[과제1] nu vs gamma 역할 비교
--------------------------------------------------------------------
    | 파라미터 | 조절 대상                    | 크게 하면                    |
    |----------|------------------------------|------------------------------|
    | nu       | 이상치 허용 비율 (전역)      | boundary 작아짐 (엄격)       |
    | gamma    | 점 영향 범위 (국소·형태)     | boundary 복잡·구불 (세밀)    |

    - nu  : "얼마나 많은 점을 이상치로 볼 것인가"
    - gamma: "경계를 얼마나 세밀하게 그릴 것인가"


--------------------------------------------------------------------
[과제1] 주요 라이브러리/함수
--------------------------------------------------------------------
- OneClassSVM(kernel="rbf", gamma=..., nu=...)
    정상 데이터만으로 이상치 탐지 모델 학습 (SVDD와 유사)

- model.fit(X_train)
    정상(Normal) 데이터만 넣어 학습 (Y 라벨 불필요)

- model.predict(X_test)
    1=정상(inlier), -1=이상치(outlier) 예측

- model.decision_function(X)
    boundary로부터의 거리 (0이 경계선, 음수=이상치 쪽)

- np.meshgrid + contour / contourf
    2D decision boundary 시각화

- f1_score(Y_test, pred, pos_label=-1)
    이상치(-1) 탐지 성능 평가


####################################################################
[2] Section 1~3 핵심 정리 (복습용)
####################################################################

One-Class Learning vs 지도학습:
    - 지도학습(2-class): Normal + Failure 라벨 모두로 학습.
    - One-Class: Normal만으로 "정상 패턴" 학습 → 벗어나면 이상.
    - Day 4-2 Resampling과 대비: 라벨된 소수 클래스 활용 vs 정상만 활용.

SVDD 개념:
    - 정상 데이터를 감싸는 최소 초구체(hypersphere) 경계 학습.
    - sklearn OneClassSVM(kernel="rbf")으로 비선형 경계 구현.

kernel 옵션 (RBF 외):
    - linear  : 직선 경계
    - poly    : 다항식 곡선
    - rbf     : 유연한 비선형 (기본, 실습 사용)
    - sigmoid : S자 형태

데이터 분할 (이 실습):
    - X_normal만 train/test split (정상만 학습)
    - test에 outlier를 합쳐 평가 (현실적 이상치 탐지 시나리오)


####################################################################
[3] 자주 헷갈리는 포인트
####################################################################

1. nu vs gamma
    - nu 작다 = boundary 넓다 (O)
    - gamma 작다 = boundary 넓고 부드럽다 (O)
    - 둘 다 "작으면 넓다"지만 조절하는 것이 다름 (비율 vs 형태).

2. gamma 크면 "원"이 아니다
    - gamma 작을 때 오히려 매끄러운 원/타원형에 가까움.
    - gamma 크면 복잡한 구불구불 형태.

3. OneClassSVM predict 라벨
    - 정상=1, 이상치=-1 (일반 이진분류 0/1과 다름).
    - F1 계산 시 pos_label=-1 로 이상치를 양성 클래스로 지정.

4. 학습에 Outlier를 넣으면 안 됨
    - One-Class는 정상만으로 boundary를 정의.
    - Outlier를 학습에 넣으면 "정상 범위"가 오염됨.

5. Day 4-2 vs Day 4-3 선택 기준
    - Failure 라벨 수백 개 + 지도학습 → Resampling (Day 4-2).
    - 정상만 충분·이상 탐지 목표 → One-Class/SVDD (Day 4-3).

6. gamma="auto"
    - 1 / (n_features * X.var()) 로 자동 설정.
    - 과제1처럼 직접 [0.01, 0.1, 1, 10] 튜닝하면 형태 변화를 명확히 볼 수 있음.

7. nu와 성능
    - nu를 무조건 크게/작게 하면 좋은 것이 아님.
    - 너무 크면 정상도 이상치 처리, 너무 작으면 이상치를 놓침.
