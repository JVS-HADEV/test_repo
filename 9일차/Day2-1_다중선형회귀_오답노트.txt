내 답변:
    1. boxplot
    2. (모름)
    3. (모름)

실제 발견한 인사이트:

    - (2) 공정한 비교 조건:

[3] 6-3) Ridge/Lasso 모델링 심화 - [과제 3] 최적 alpha 탐색
####################################################################

Q3. (1) 파라미터가 적고 후보 값도 적을 때 사용하는 하이퍼파라미터 탐색
        방법은?
    (2) alpha 후보를 등차수열이 아닌 로그 스케일(등비수열)로 넓게 잡는
        이유는?

내 답변:
    1. GridSearchCV
    2. alpha에 따른 결과가 지수함수적으로 감소하기 때문?

채점: (1) 정답 / (2) 부분 오답 (설명이 부정확)

해설:
    - (1) 정답.
    - (2) "결과가 지수적으로 감소한다"가 아니라, alpha가 모델(규제 강도)에
      미치는 영향의 "체감 크기"가 절대적 차이가 아니라 배수(자릿수)
      차이에 좌우되기 때문. 예를 들어 alpha를 1->11로 늘리는 것보다
      0.1->1로 늘리는 것(둘 다 앞자리 기준 10배)이 규제 강도에 더 비슷한
      "정도"의 변화를 줌. 그래서 [0.001, 0.01, 0.1, 1, 10, 100]처럼
      로그 스케일로 잡아야 작은 값부터 큰 값까지 고르게 탐색 가능.

실제 결과:
    - Ridge 최적 alpha: 100 (R² 0.635, 기존 alpha=0.1/10과 거의 동일)
    - Lasso 최적 alpha: 0.001 (R² 0.635, 역시 큰 차이 없음)
    - 이 데이터셋은 규제 강도 조정만으로는 성능이 크게 개선되지 않음

[4] 6-3) Ridge/Lasso 모델링 심화 - [과제 4] ElasticNet
####################################################################

Q4. (1) L1(Lasso)과 L2(Ridge) 규제를 동시에 활용하는 모델의 이름은?
    (2) 이 모델에서 L1 비중을 조절하는 하이퍼파라미터 이름은?

내 답변:
    1. ridge-lasso
    2. (모름)

채점: (1) 개념은 맞으나 명칭 오답 / (2) 모름 -> 정답 안내

해설:
    - (1) 정답은 ElasticNet. Ridge와 Lasso를 합친 모델이라는 이해는
      정확했지만, 정식 명칭이 다름.
    - (2) l1_ratio (0~1 사이 값)
        * l1_ratio=1 -> Lasso와 동일
        * l1_ratio=0 -> Ridge와 동일
        * l1_ratio=0.5 -> L1/L2를 절반씩 섞음

실제 결과:
    - 최적 파라미터: alpha=0.1, l1_ratio=0.1
    - 성능은 Ridge/Lasso와 유사한 수준 (R² 약 0.635)


####################################################################
[5] 6-3) Ridge/Lasso 모델링 심화 - [과제 5] Age 비선형성 반영
####################################################################

Q5. (1) 선형회귀가 "처음엔 급증, 나중엔 완만"한 비선형 관계를 반영하려면
        어떤 방법을 쓸 수 있는가?
    (2) 로그 변환이 왜 이런 관계를 잘 표현하는가?

내 답변:
    1. 변수에 log를 취한다
    2. log 함수가 처음엔 빠르고 나중엔 완만한 곡선의 형태를 취하기 때문

채점: (1) 정답 / (2) 정답

해설:
    - log 함수는 입력이 작을 때 출력이 급격히 변하고, 입력이 커질수록
      변화 폭이 줄어드는 오목(concave) 함수. Age를 log(Age)로 바꾸면
      선형회귀는 여전히 "직선" 관계 하나만 학습하지만, 그 직선이
      log(Age) 축 위에 그려지기 때문에 원래 Age 축에서 보면 곡선처럼
      보이는 효과를 얻음.

실제 결과 (가장 큰 성능 개선!):
    - 로그 변환 전: RMSE 9.676, R² 0.635
    - 로그 변환 후: RMSE 6.577, R² 0.831
    -> [과제 3], [과제 4]의 규제 강도 조정보다 훨씬 큰 효과.
       "모델을 바꾸는 것"보다 "변수의 형태를 실제 관계에 맞게 다듬는 것"이
       더 큰 성능 개선을 가져올 수 있음을 보여주는 사례.


====================================================================
 과제별 주요 라이브러리 / 함수 설명
====================================================================

- seaborn.boxplot(y=series)
    하나의 변수에 대한 중앙값/사분위수/이상치를 박스와 수염으로 시각화.

- seaborn.scatterplot(x, y) / plt.scatter(x, y)
    두 변수 간의 관계를 점으로 표시. 선형/비선형 등 관계의 "형태"를
    직접 눈으로 확인할 수 있음.

- seaborn.pairplot(df)
    데이터프레임의 모든 수치형 변수 쌍에 대한 산점도를 격자로 한 번에
    시각화. 대각선에는 각 변수의 분포(히스토그램)가 표시됨.

- pandas 파생변수 생성 (사칙연산)
    df["new_col"] = df["colA"] / df["colB"] 처럼 기존 컬럼들을 연산해
    새로운 의미를 가진 변수를 생성. 도메인 지식(예: 물-시멘트비)을
    반영할 때 유용.

- sklearn.model_selection.GridSearchCV(estimator, param_grid, cv, scoring)
    지정한 하이퍼파라미터 값의 모든 조합을 교차검증하며 최적 조합 탐색.
    파라미터 개수가 적고 후보 값도 적을 때 적합.
    .best_params_ : 최적 파라미터 조합
    .best_estimator_ : 최적 파라미터로 재학습된 모델

- sklearn.linear_model.ElasticNet(alpha, l1_ratio)
    L1(Lasso) 규제와 L2(Ridge) 규제를 함께 사용하는 선형 모델.
    alpha: 전체 규제 강도, l1_ratio: L1 규제의 비중(0~1)

- numpy.log(x)
    자연로그 변환. 오른쪽으로 꼬리가 길게 늘어진(right-skewed) 분포나,
    "처음엔 급증 후 완만해지는" 비선형 관계를 선형 모델에 반영할 때 사용.
    주의: x는 반드시 양수여야 함 (0 이하 값이 있으면 에러/결측 발생).

- get_regscore(true, pred)  [노트북 내 사용자 정의 함수]
    회귀 모델의 MSE, RMSE, MAE, R-squared를 한 번에 출력하는 헬퍼 함수.
